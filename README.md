# ГЭК: приказы, протоколы, отчёты — Streamlit-приложение

Единый интерфейс для всех генераторов документов ГЭК, ранее живших как отдельные `.ipynb`.
Все данные читаются из [Google-таблицы «ВКР UU»](https://docs.google.com/spreadsheets/d/1bvZ4C-3Mf7-sRmpL7F1N_xUT4NfXzDPNicLGOQ1_a7M/edit).

## Запуск

```bash
cd streamlit_app
pip install -r requirements.txt
streamlit run app.py
```

Приложение ожидает, что в корне репозитория (рядом с `logic/`) лежат папки `pics/` и `fonts/` —
логотип и шрифты для PDF. Менять их местоположение не нужно, `logic/common.py` находит их
автоматически по расположению самого файла.

## Что внутри

```
streamlit_app/
  app.py                  # главная страница, ссылки на все генераторы
  ui_helpers.py            # источник данных (сайдбар), скачивание PDF/zip, лог
  logic/                   # чистая логика генерации (без Streamlit), можно использовать из CLI
    common.py               # доступ к Sheets, Typst-компиляция, даты, общая шапка приказов
    prikaz_topics.py         # приказ: темы и руководители ВКР
    dopusk_gia.py            # приказ: допуск к ГИА
    prikaz_gek.py            # приказ: состав и секретари ГЭК
    prikaz_schedule.py       # приказ: расписание ГИА
    prikaz_appeal.py         # приказ: апелляционная комиссия
    protocol.py               # протоколы заседаний ГЭК
    report.py                 # отчёт председателя ГЭК
    reviews.py                 # отзывы и рецензии на ВКР
    statements.py              # заявления студентов (тема ВКР / ГИА / каникулы)
  pages/
    0_Сгенерировать_всё.py    # прогоняет все генераторы разом, один общий zip
    1..9_*.py                  # по странице на каждый генератор/тип документа
```

## Генерация «всё» vs «выборочно»

У каждого генератора (кроме апелляционной комиссии — она одна на всех направлений) есть
блок **«🎯 Что генерировать — всё или выборочно»**: три мультиселекта (факультет / направление /
программа). Пустой выбор = без фильтра = генерируется для всех. Логика фильтрации — в
`logic/common.py: filter_students()`, применяется в каждом `generate()` до группировки данных.

Страница **«Сгенерировать всё»** прогоняет все включённые типы документов за один проход с
общим фильтром и общими датами, складывает результат в один `.zip` с подпапками по типам
документов (`01_Приказ_темы_и_руководители/`, `06_Протоколы_ГЭК/`, `09_Заявления/...` и т.д.).

## Источники данных (вкладки таблицы)

| Вкладка | Что там | Кто использует |
|---|---|---|
| `2025/26` | обучающиеся: ФИО, курс, направление, программа, тема, руководитель, рецензент, оценка, диплом с отличием | все генераторы |
| `ГЭК` | состав комиссий (председатель/члены/секретарь) по факультету+программе | приказ о составе ГЭК, протоколы, отчёт |
| `График` | даты защит (этап «защита…», аудитория в «примечание») | приказ о расписании ГИА, протоколы, отчёт |
| `Руководители`, `Рецензенты` | справочные списки (выпадающие списки в основной таблице) | — |
| `Реестр файлов` | id таблиц отзывов/рецензий, которые создаёт `generate_review_tables.gs` | страница «Отзывы и рецензии» |

## Отзывы и рецензии — особый случай

Тексты отзывов/рецензий заполняются не в этом приложении, а в отдельных таблицах, которые
создаёт Apps Script `finals/archive/generate_review_tables.gs` (меню **«ВКР»** внутри самой
Google-таблицы). Страница «Отзывы и рецензии» только собирает финальные PDF по данным из
этих таблиц (через вкладку «Реестр файлов»).

## Известные исправления при консолидации

- **Допуск к ГИА**: в исходном ноутбуке был жёстко зашит фильтр «только 4 курс», что молча
  исключало архитектуру (5 курс) и магистратуру (2 курс) из приказа. Теперь курсы выбираются
  явно в UI.
- Все асеты (`image("/pics/logo.svg")`) теперь используют единый абсолютный путь от `root`,
  а не смесь относительных путей — раньше `generate_prikaz.ipynb` и `dopusk GIA` полагались
  на текущую рабочую директорию, что ломалось при запуске не из `Current/`.
